
# AWS的S3服务如何获取Access key ID和Secret access key - 高科的专栏 - CSDN博客

In this post, we will explain how to retrieve Amazon Access Key ID and Secret Access Key. These keys allow you to control your AWS account.We'll focus on the following topics:
How to Retrieve Root Access Keys
How to Retrieve IAM Access Keys
Note:As a best practice, AWS recommends creating an IAM user rather than relying on root access keys. IAM access keys allow you to securely control access to AWS services and resources for your users.
Below, please find instructions on how to retrieve root access keys and IAM access keys.
![Test your AWS knowledge CTA](http://www.cloudberrylab.com/blog/wp-content/uploads/2015/08/but-1.png)
## How to Retrieve Root Access Keys

Go to[Amazon Web Services](https://www.amazon.com/ap/signin?openid.assoc_handle=aws&openid.return_to=https%3A%2F%2Fsignin.aws.amazon.com%2Foauth%3Fresponse_type%3Dcode%26client_id%3Darn%253Aaws%253Aiam%253A%253A015428540659%253Auser%252Fhomepage%26redirect_uri%3Dhttps%253A%252F%252Fconsole.aws.amazon.com%252Fconsole%252Fhome%253Fstate%253DhashArgs%252523%2526isauthcode%253Dtrue%26noAuthCookie%3Dtrue&openid.mode=checkid_setup&openid.ns=http%3A%2F%2Fspecs.openid.net%2Fauth%2F2.0&openid.identity=http%3A%2F%2Fspecs.openid.net%2Fauth%2F2.0%2Fidentifier_select&openid.claimed_id=http%3A%2F%2Fspecs.openid.net%2Fauth%2F2.0%2Fidentifier_select&action=&disableCorpSignUp=&clientContext=&marketPlaceId=&poolName=&authCookies=&pageId=aws.ssop&siteState=registered%2Cen_US&accountStatusPolicy=P1&sso=&openid.pape.preferred_auth_policies=MultifactorPhysical&openid.pape.max_auth_age=120&openid.ns.pape=http%3A%2F%2Fspecs.openid.net%2Fextensions%2Fpape%2F1.0&server=%2Fap%2Fsignin%3Fie%3DUTF8&accountPoolAlias=&forceMobileApp=0&language=en_US&forceMobileLayout=0)console andclick on the name of your account (it is located in the top right corner of the console). Then, in the expanded drop-down list, selectSecurity Credentials.![Retrieving AWS Root Access Key Step 1](https://www.cloudberrylab.com/blog/wp-content/uploads/2015/05/11.png)2. Click theContinue to Security Credentialsbutton.![Retrieving AWS Root Access Key Step 2](https://www.cloudberrylab.com/blog/wp-content/uploads/2015/05/2.5.png)3. Expand theAccess Keys (Access Key ID and Secret Access Key)option. You will see the list of your active and deleted access keys.![Retrieving AWS Root Access Key Step 3](https://www.cloudberrylab.com/blog/wp-content/uploads/2015/06/2015-06-23-21_05_02-IAM-Management-Console.png)Note:You cannot retrieve the existing secret key. You can see the secret key only once immediately after creating. So, in order to get a secret key, you will need to create a new one.
4. To generate new access keys, click theCreate New Access Keybutton.![Retrieving AWS Root Access Key Step 4](https://www.cloudberrylab.com/blog/wp-content/uploads/2015/06/create_new_access_key1.png)5. ClickShow Access Keyto have it displayed on the screen. Note, that you can download it to your machine as a file and open it whenever needed. To download it, just click theDownload Key Filebutton.![Retrieving AWS Root Access Key Step 5](https://www.cloudberrylab.com/blog/wp-content/uploads/2010/03/signUpS3screen6.png)Attention!If you do not write down the key or download the key file to your computer before you press "Close" or "Cancel" you will not be able to retrieve the secret key in future. Then you'll have to delete the keys which you created and start to create new keys.
## How to Retrieve IAM Access Keys
1. Go to[Amazon Web Services](https://www.amazon.com/ap/signin?openid.assoc_handle=aws&openid.return_to=https%3A%2F%2Fsignin.aws.amazon.com%2Foauth%3Fresponse_type%3Dcode%26client_id%3Darn%253Aaws%253Aiam%253A%253A015428540659%253Auser%252Fhomepage%26redirect_uri%3Dhttps%253A%252F%252Fconsole.aws.amazon.com%252Fconsole%252Fhome%253Fstate%253DhashArgs%252523%2526isauthcode%253Dtrue%26noAuthCookie%3Dtrue&openid.mode=checkid_setup&openid.ns=http%3A%2F%2Fspecs.openid.net%2Fauth%2F2.0&openid.identity=http%3A%2F%2Fspecs.openid.net%2Fauth%2F2.0%2Fidentifier_select&openid.claimed_id=http%3A%2F%2Fspecs.openid.net%2Fauth%2F2.0%2Fidentifier_select&action=&disableCorpSignUp=&clientContext=&marketPlaceId=&poolName=&authCookies=&pageId=aws.ssop&siteState=registered%2Cen_US&accountStatusPolicy=P1&sso=&openid.pape.preferred_auth_policies=MultifactorPhysical&openid.pape.max_auth_age=120&openid.ns.pape=http%3A%2F%2Fspecs.openid.net%2Fextensions%2Fpape%2F1.0&server=%2Fap%2Fsignin%3Fie%3DUTF8&accountPoolAlias=&forceMobileApp=0&language=en_US&forceMobileLayout=0)consoleand click on the name of your account (it is located in the top right corner of the console). In the expanded drop-down list, selectSecurity Credentials.![Retrieving AWS IAM Access Key Step 1](https://www.cloudberrylab.com/blog/wp-content/uploads/2015/05/11.png)2. Click theGet Started with IAM Usersbutton.![Retrieving AWS IAM Access Key Step 2](https://www.cloudberrylab.com/blog/wp-content/uploads/2015/06/2015-06-01-13_55_24-IAM-Management-Console.png)3. This will bring you to theIAM Dashboardwhere you can manage IAM users and their permissions such as creating new IAM users, adding IAM users to the custom groups, granting them a certain level of permissions, etc.![Retrieving AWS IAM Access Key Step 3](https://www.cloudberrylab.com/blog/wp-content/uploads/2015/06/56.png)4. To review the IAM access keys, click on a particular user.![Retrieving AWS IAM Access Key Step 4](https://www.cloudberrylab.com/blog/wp-content/uploads/2010/03/screen2.png)5. UnderSecurity Credentials, you will see a list of Access Keys for the IAM user.![Retrieving AWS IAM Access Key Step 5](https://www.cloudberrylab.com/blog/wp-content/uploads/2010/03/screen3.png)Note:You cannot retrieve the existing secret keys. You can see the secret key only once immediately after creating. So, in order to get a secret key, you will need to create a new one.
6. ClickCreate Access Keyto create a new key. The new access keys will be generated and displayed on the screen.![Retrieving AWS IAM Access Key Step 6](https://www.cloudberrylab.com/blog/wp-content/uploads/2010/03/screen4.png)Attention!If you do not write down the key or download the key file to your computer before you press "Close" or "Cancel" you will not be able to retrieve the secret key in future. Then you'll have to delete the keys which you created start to create new keys.
